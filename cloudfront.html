⚡ Amazon CloudFront

Content Delivery Network Edge Service AWS Solutions Architect Associate High Exam Importance
Overview Features Core Concepts Security Architecture Use Cases Exam Notes Summary

Service Overview

Amazon CloudFront is a fast content delivery network (CDN) service that securely delivers data, videos, applications and APIs to customers globally with low latency and high transfer speeds.

CloudFront uses a global network of edge locations to cache content closer to end users, reducing the load on origin servers and dramatically improving performance.

Key Features

  • Global edge network with 400+ points of presence in 90+ cities
  • Low latency delivery by caching content at edge locations
  • HTTPS support with free SSL/TLS certificates via AWS Certificate Manager
  • DDoS protection integrated with AWS Shield Standard (free)
  • Integration with AWS WAF for web application firewall rules
  • Signed URLs and Signed Cookies for restricting access to private content
  • Lambda@Edge — run code at edge locations to customize content delivery

Core Concepts

Distribution

A CloudFront distribution is the configuration that tells CloudFront where your origin is and how to deliver content. There are two types: Web distributions (for websites and APIs) and RTMP (legacy streaming).

Origins

An origin is the source of the content that CloudFront distributes. Origins can be an S3 bucket, an EC2 instance, an Elastic Load Balancer, or any custom HTTP server.

Edge Locations

Edge Locations are the data centers in CloudFront's global network where content is cached. When a user requests content, CloudFront routes the request to the nearest edge location.

Cache Behavior & TTL

Cache behaviors define how CloudFront handles requests for different URL patterns. TTL (Time to Live) controls how long objects are cached before CloudFront checks the origin for a new version.

Invalidation

Cache invalidation removes objects from edge caches before their TTL expires. Useful after deploying updated website content.

Security Considerations

  • Use HTTPS for all distributions — enforce redirect from HTTP to HTTPS
  • Signed URLs — grant time-limited access to individual files
  • Signed Cookies — grant time-limited access to multiple files
  • Use Origin Access Control (OAC) to restrict S3 bucket access to CloudFront only
  • Integrate AWS WAF to block common web exploits at the edge
  • AWS Shield Standard is automatically included with CloudFront

Real-World Use Cases

  • Static website acceleration — serve HTML/CSS/JS from S3 via CloudFront globally
  • Video streaming — deliver on-demand or live video with low latency
  • API acceleration — cache API responses at the edge to reduce backend load
  • Software distribution — deliver large installer files or app updates globally
  • Secure content delivery — stream premium content using Signed URLs

AWS Solutions Architect Exam Notes

  • CloudFront is a global service — not region-specific
  • There are more Edge Locations than Regions or Availability Zones
  • Use Origin Access Control (OAC) to ensure S3 content is only served via CloudFront
  • CloudFront reduces latency by serving cached content from the nearest edge location
  • Signed URLs are for single files; Signed Cookies are for multiple files
  • AWS Shield Standard DDoS protection is automatically included at no extra cost
  • Cache invalidation costs money — use versioned file names instead where possible
  • CloudFront integrates with ACM for free SSL/TLS certificates (must be in us-east-1)

Key Takeaways

  • CloudFront is AWS's CDN — it caches content at edge locations worldwide
  • Origins can be S3, EC2, ALB or any HTTP server
  • Use OAC to lock down S3 origins so only CloudFront can access them
  • Signed URLs and Cookies restrict access to private or premium content
  • DDoS protection via Shield Standard is included automatically
