📊 Amazon CloudWatch

Monitoring & Observability Core AWS Service AWS Solutions Architect Associate Very High Exam Importance
Overview Metrics Logs Alarms Events & Dashboards Security Architecture Exam Notes Summary

Service Overview

Amazon CloudWatch is a monitoring and observability service for AWS cloud resources and applications. It collects metrics, logs and events, and gives you a unified view of operational health. CloudWatch enables you to set alarms, visualise logs and metrics, and automatically respond to changes.

CloudWatch Metrics

Metrics are time-series data points from AWS services and your applications. AWS services automatically publish metrics to CloudWatch — no configuration needed.

Important Default Metrics

  • EC2: CPUUtilization, NetworkIn/Out, DiskReadOps, StatusCheckFailed
  • EBS: VolumeReadOps, VolumeWriteOps, BurstBalance
  • RDS: CPUUtilization, FreeableMemory, DatabaseConnections, ReadLatency
  • Lambda: Invocations, Duration, Errors, Throttles, ConcurrentExecutions
  • ALB: RequestCount, TargetResponseTime, HTTP 4XX/5XX errors
  • SQS: NumberOfMessagesSent, ApproximateAgeOfOldestMessage

EC2 Metrics NOT Available by Default

By default, EC2 does not publish memory utilisation, disk space or swap metrics to CloudWatch — these require the CloudWatch Agent installed on the instance.

Metric Resolution

  • Standard resolution — 1-minute granularity (free for most services)
  • High resolution — 1-second granularity (custom metrics, higher cost)

CloudWatch Logs

CloudWatch Logs centralises log data from EC2 instances, Lambda functions, ECS containers, CloudTrail, VPC Flow Logs and more.

Key Concepts

  • Log Group — container for log streams from the same source (e.g. /aws/lambda/my-function)
  • Log Stream — sequence of log events from a single source (one EC2 instance, one Lambda execution)
  • Retention policy — set log retention from 1 day to never expire (default: never expire)
  • CloudWatch Logs Insights — query log data using SQL-like syntax for fast analysis
  • Metric Filters — extract metric data from log entries (e.g. count ERROR occurrences per minute)

CloudWatch Agent

Install the CloudWatch Agent on EC2 instances to collect: memory usage, disk space, swap, custom application logs. These are not available in CloudWatch by default.

CloudWatch Alarms

Alarms watch a single metric over a time period and perform actions when the metric crosses a threshold.

Alarm States

  • OK — metric within threshold
  • ALARM — metric breached threshold
  • INSUFFICIENT_DATA — not enough data to determine state

Alarm Actions

  • Send notification via SNS → email/SMS/Lambda
  • Trigger Auto Scaling actions
  • Stop, terminate, reboot or recover EC2 instances
  • Trigger Systems Manager OpsItems

Composite Alarms

Composite alarms combine multiple alarms using AND/OR logic. Useful for reducing alarm noise — only alert when multiple conditions are met simultaneously.

CloudWatch Events & Dashboards

CloudWatch Events (now EventBridge)

CloudWatch Events delivers a stream of real-time events from AWS services. This is now part of Amazon EventBridge — the same functionality with more features.

CloudWatch Dashboards

  • Create custom visualisations of metrics and alarms
  • Dashboards are global — you can display metrics from multiple regions
  • Share dashboards with stakeholders without giving AWS console access

Security Considerations

  • Use IAM policies to control who can view metrics, logs and alarms
  • Encrypt log data at rest using KMS
  • Set log retention policies to avoid unnecessary storage costs
  • Use CloudWatch Logs Insights for fast log analysis without exporting data
  • Forward logs to S3 for long-term archiving via subscription filters

Architecture Overview

AWS Services (EC2, RDS, Lambda, ALB...)
        │
        ▼
CloudWatch Metrics + CloudWatch Logs
        │
        ├── Dashboards (visualise health)
        │
        ├── CloudWatch Alarm
        │         │
        │    ┌────┴────────────┐
        │    SNS Notification   Auto Scaling Action
        │    (email/SMS/Lambda) (add/remove instances)
        │
        └── Logs Insights (query and analyse logs)

AWS Solutions Architect Exam Notes

  • CloudWatch = monitoring and observability; CloudTrail = API audit logging
  • EC2 does NOT report memory/disk metrics by default — install CloudWatch Agent
  • Standard metrics = 1-minute resolution; High-resolution = 1-second
  • CloudWatch Alarms trigger SNS, Auto Scaling, or EC2 actions
  • Composite alarms combine multiple alarms with AND/OR — reduces alert noise
  • CloudWatch Logs Insights = fast log querying — no need to export to Athena for quick analysis
  • Metric Filters convert log entries into CloudWatch metrics — e.g. count HTTP 500 errors
  • Dashboards are global — display metrics from multiple regions in one view
  • Log retention defaults to never expire — always set a retention policy to control costs

Key Takeaways

  • CloudWatch is the central monitoring service for all AWS resources
  • Metrics, Logs, Alarms and Dashboards give full operational visibility
  • Install CloudWatch Agent for memory, disk and custom application metrics from EC2
  • Alarms trigger automated responses — scale, notify, recover
  • Use Logs Insights for fast ad-hoc analysis of log data without exporting
