🔗 Direct Connect, VPN & Transit Gateway

Hybrid Connectivity Networking AWS Solutions Architect Associate High Exam Importance
Direct Connect Site-to-Site VPN Transit Gateway Comparison Architecture Exam Notes Summary

AWS Direct Connect

AWS Direct Connect establishes a dedicated, private network connection from your on-premises data centre to AWS. Traffic travels over a private leased line — not over the public internet.

Key Features

  • Dedicated private connection — not shared with internet traffic
  • Consistent network performance — no internet congestion or variable latency
  • Available in 1Gbps, 10Gbps and 100Gbps connection speeds
  • Reduces bandwidth costs for large data transfers
  • Hosted connections available through AWS partners for smaller bandwidths (50Mbps–10Gbps)
  • Takes weeks to months to provision — not for quick setup

Direct Connect Gateway

A Direct Connect Gateway allows a single Direct Connect connection to access VPCs in multiple AWS regions globally — avoiding the need for a separate connection per region.

Resilient Direct Connect Setup

  • Development/Test — Single Direct Connect + Site-to-Site VPN as backup
  • High availability — Two Direct Connect connections in different locations
  • Maximum resilience — Two connections at two separate locations from two different providers

AWS Site-to-Site VPN

AWS Site-to-Site VPN creates an encrypted IPSec tunnel between your on-premises network and your AWS VPC over the public internet.

Components

  • Virtual Private Gateway (VGW) — the AWS side of the VPN tunnel, attached to your VPC
  • Customer Gateway (CGW) — represents your on-premises VPN device in AWS
  • Each VPN connection creates two tunnels for redundancy

AWS VPN CloudHub

VPN CloudHub enables multiple on-premises sites to connect to a single Virtual Private Gateway and communicate with each other through AWS — creating a hub-and-spoke VPN network.

VPN vs Direct Connect

Feature           Site-to-Site VPN    Direct Connect
──────────────────────────────────────────────────────
Setup time        Minutes/Hours       Weeks/Months
Cost              Low                 Higher
Bandwidth         Limited (~1.25Gbps) Up to 100Gbps
Consistency       Variable (internet) Consistent (private)
Encryption        IPSec (built-in)    Not encrypted by default (use MACsec)
Public internet   ✅ Uses internet    ❌ Private line
Use case          Quick setup, backup High throughput, compliance

AWS Transit Gateway

Transit Gateway is a highly scalable network transit hub that connects thousands of VPCs and on-premises networks through a single gateway. It solves the problem of complex VPC peering at scale.

Why Transit Gateway?

VPC Peering is non-transitive — if VPC A peers with VPC B and VPC B peers with VPC C, VPC A cannot communicate with VPC C. Managing hundreds of peering connections becomes unmanageable. Transit Gateway acts as a central hub — all VPCs connect to it and can communicate through it.

Key Features

  • Connect up to 5,000 VPCs and on-premises networks
  • Transitive routing — all attached networks can communicate
  • Attach VPCs, Direct Connect gateways, VPN connections, and SD-WAN
  • Route tables — control which networks can communicate with each other
  • Supports inter-region peering between Transit Gateways
  • Multicast support for streaming applications

Architecture Overview

Without Transit Gateway (complex):
VPC-A ←→ VPC-B ←→ VPC-C
  ↕                  ↕
VPC-D ←→ VPC-E ←→ VPC-F
(15 peering connections for 6 VPCs)

With Transit Gateway (simple hub-and-spoke):
        Transit Gateway
        ┌──────┬──────┐
    VPC-A  VPC-B  VPC-C
    VPC-D  VPC-E  On-premises (Direct Connect/VPN)
(All connect through one hub)

AWS Solutions Architect Exam Notes

  • Direct Connect = private dedicated line, consistent, high bandwidth, weeks to set up
  • Site-to-Site VPN = IPSec over internet, quick setup, variable performance
  • Best of both = Direct Connect as primary + VPN as backup failover
  • Transit Gateway = hub for connecting many VPCs and on-premises — replaces complex peering meshes
  • VPC Peering is non-transitive — Transit Gateway solves this
  • Direct Connect does not encrypt by default — combine with VPN for encryption over Direct Connect
  • VPN CloudHub = multiple on-premises sites connected via same VGW for site-to-site communication
  • For compliance/regulated data → use Direct Connect (avoids public internet)
  • For quick hybrid connectivity → use Site-to-Site VPN

Key Takeaways

  • Direct Connect = private dedicated line for consistent, high-throughput hybrid connectivity
  • Site-to-Site VPN = fast, cheap, encrypted tunnel over the internet — good as backup
  • Transit Gateway = central hub for large-scale VPC and on-premises connectivity
  • Combine Direct Connect + VPN for redundancy and encryption
  • Use Transit Gateway instead of complex VPC peering meshes at scale
