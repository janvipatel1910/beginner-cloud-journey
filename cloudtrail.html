🔍 AWS CloudTrail

Audit & Compliance Governance AWS Solutions Architect Associate High Exam Importance
Overview Features Core Concepts Security Architecture Use Cases Exam Notes Summary

Service Overview

AWS CloudTrail is a service that enables governance, compliance, operational auditing and risk auditing of your AWS account. CloudTrail records API calls made in your account — who did what, when, and from where.

CloudTrail is enabled by default in all AWS accounts and retains 90 days of management event history at no charge.

Key Features

  • Records all API calls made via AWS Console, CLI, SDK and services
  • Captures who made the request, what was requested, when, and from which IP
  • 90 days of event history by default — free of charge
  • Create Trails to store logs in S3 for longer retention
  • Integrate with CloudWatch Logs for real-time monitoring and alerting
  • CloudTrail Insights — automatically detect unusual API activity
  • Multi-region and organisation-wide trails
  • Log file validation — detect tampering with log files

Core Concepts

Event Types

  • Management Events — control plane operations: creating/deleting resources, IAM changes, VPC configuration. Logged by default.
  • Data Events — data plane operations: S3 object-level access (GetObject, PutObject), Lambda function invocations. Not logged by default — must be explicitly enabled (additional cost).
  • Insights Events — unusual write API activity patterns detected by CloudTrail Insights.

Trails

A trail is a configuration that delivers CloudTrail events to an S3 bucket. Without a trail, only 90 days of history are kept. Create a trail for:

  • Retention beyond 90 days
  • Sending logs to CloudWatch Logs for alerts
  • Covering all regions (multi-region trail)
  • Organisation-wide logging across all accounts

CloudTrail vs CloudWatch

  • CloudTrail — answers "who did what in my AWS account?" — API-level audit log
  • CloudWatch — answers "how is my application performing?" — metrics, logs, alarms
  • They work together: CloudTrail logs → CloudWatch Logs → CloudWatch Alarms → SNS notification

Security Considerations

  • Enable a multi-region trail to capture events across all regions
  • Send CloudTrail logs to a dedicated S3 bucket in a separate security account
  • Enable log file validation to detect if log files have been tampered with
  • Enable S3 MFA Delete on the log bucket to prevent log deletion
  • Enable CloudTrail Insights to detect unusual activity patterns
  • Use CloudTrail + CloudWatch + SNS for real-time security alerting

Real-World Use Cases

  • Security investigation — identify who deleted an S3 bucket or modified a security group
  • Compliance — provide audit logs for SOC2, PCI-DSS, HIPAA, ISO 27001
  • Real-time alerting — detect and alert on root account logins or IAM policy changes
  • Forensic analysis — reconstruct events leading up to a security incident
  • Cost allocation — track which users are creating expensive resources

AWS Solutions Architect Exam Notes

  • CloudTrail is enabled by default in all AWS accounts
  • Default retention is 90 days — create a Trail for longer storage in S3
  • Management events are logged by default; data events must be explicitly enabled
  • If asked "who deleted/changed X in AWS" → the answer is always CloudTrail
  • CloudTrail logs are stored in S3 — use S3 lifecycle policies to move old logs to Glacier
  • Enable log file validation to detect if logs have been tampered with after delivery
  • CloudTrail ≠ CloudWatch — CloudTrail is for API auditing, CloudWatch is for performance monitoring
  • For organisation-wide logging: create a trail in the management account that applies to all member accounts

Key Takeaways

  • CloudTrail records every API call in your AWS account — who, what, when, where
  • Create a Trail to store logs in S3 beyond 90 days and enable real-time alerting
  • Use CloudTrail + CloudWatch Logs + SNS for a complete security monitoring pipeline
  • Enable log file validation and store logs in a separate security account
  • CloudTrail is the foundation of audit and compliance in AWS
