🌍 AWS Global Accelerator

Networking Service Global Performance AWS Solutions Architect Associate High Exam Importance
Overview Features Core Concepts vs CloudFront Architecture Exam Notes Summary

Service Overview

AWS Global Accelerator is a networking service that improves the availability and performance of your applications for global users. It routes traffic through AWS's private global network instead of the public internet, reducing latency and improving reliability.

Global Accelerator provides you with two static Anycast IP addresses that act as a global entry point to your application endpoints in multiple AWS regions.

Key Features

  • Two static Anycast IP addresses — fixed entry points for your application
  • Routes traffic over AWS's private global backbone network
  • Continuous health checks — automatically routes away from unhealthy endpoints
  • Instant failover — traffic rerouted in seconds when an endpoint becomes unhealthy
  • Traffic dials — control the percentage of traffic sent to each endpoint group
  • Works with: ALB, NLB, EC2 instances, Elastic IPs
  • DDoS protection via AWS Shield Standard

Core Concepts

Anycast IP Addresses

Global Accelerator assigns two static Anycast IP addresses to your accelerator. Anycast means that globally distributed edge locations all advertise the same IP. User traffic is automatically routed to the nearest edge location, which then forwards traffic to your backend over AWS's private network.

Endpoint Groups

An endpoint group is associated with an AWS region. Each group contains one or more endpoints (ALB, NLB, EC2, EIP). You can assign traffic dials (0–100%) to control how much traffic goes to each region.

Health Checks

Global Accelerator continuously monitors endpoint health. If an endpoint or region becomes unhealthy, traffic is automatically rerouted to the next healthy endpoint — typically within 30 seconds.

Global Accelerator vs CloudFront

Feature                  Global Accelerator        CloudFront
──────────────────────────────────────────────────────────────────
Primary use              TCP/UDP performance       HTTP/HTTPS content delivery
Caching                  ❌ No                     ✅ Yes (CDN)
Static IPs               ✅ 2 Anycast IPs          ❌ No (domain-based)
Works with non-HTTP       ✅ Yes (TCP, UDP)         ❌ No (HTTP/S only)
Failover speed           Seconds                   Minutes (DNS TTL)
Gaming / VoIP            ✅ Ideal                  ❌ Not suitable
Content delivery         ❌ Not caching             ✅ Ideal
Protocols                TCP, UDP                  HTTP, HTTPS, WebSockets

Rule of thumb: Use CloudFront for HTTP content caching and delivery. Use Global Accelerator for TCP/UDP performance, static IPs, or non-HTTP protocols.

Architecture Overview

User (anywhere in the world)
        │
        ▼
AWS Edge Location (nearest Anycast entry point)
        │
        ▼
AWS Global Private Network (backbone)
        │
        ├── us-east-1 Endpoint Group (ALB) — 50% traffic
        │
        └── eu-west-1 Endpoint Group (ALB) — 50% traffic

If us-east-1 fails → 100% traffic automatically routed to eu-west-1

AWS Solutions Architect Exam Notes

  • Global Accelerator uses Anycast static IPs — CloudFront uses domain names (no static IPs)
  • Global Accelerator routes over AWS private network — CloudFront serves from edge cache
  • Use Global Accelerator for: non-HTTP protocols (TCP/UDP), static IPs required, gaming, IoT, VoIP
  • Use CloudFront for: HTTP/S content caching and delivery
  • Global Accelerator performs health checks and provides instant failover (seconds vs DNS TTL minutes)
  • Both services use AWS edge locations as entry points — but serve different purposes
  • Global Accelerator ≠ CDN — it does not cache content

Key Takeaways

  • Global Accelerator improves global application performance by routing traffic through AWS's private network
  • Two static Anycast IPs — simple DNS setup, stable entry points
  • Instant failover based on health checks — more reliable than DNS-based failover
  • Use for non-HTTP applications, static IPs, or ultra-low latency global access
  • CloudFront = content caching; Global Accelerator = network path optimisation
