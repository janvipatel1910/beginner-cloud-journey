📁 Amazon Elastic File System (EFS)

Storage Service Shared File Storage AWS Solutions Architect Associate Medium Exam Importance
Overview Features Storage Classes EFS vs EBS vs S3 Security Architecture Exam Notes Summary

Service Overview

Amazon Elastic File System (EFS) is a fully managed, scalable shared file system for use with AWS cloud services and on-premises resources. EFS automatically grows and shrinks as you add or remove files — you never need to provision capacity in advance.

EFS is a Network File System (NFS) — multiple EC2 instances across multiple Availability Zones can read and write to the same EFS file system simultaneously.

Key Features

  • Shared access — thousands of EC2 instances can access simultaneously
  • Automatically scales from gigabytes to petabytes with no pre-provisioning
  • Multi-AZ — data is stored redundantly across multiple Availability Zones
  • Compatible with Linux-based workloads (NFS protocol)
  • Supports both regional and One Zone storage classes
  • Lifecycle management — automatically move infrequent files to cheaper storage
  • Encryption at rest (KMS) and in transit (TLS)

EFS Storage Classes

  • EFS Standard — for frequently accessed files, multi-AZ redundancy
  • EFS Standard-IA (Infrequent Access) — lower cost for files not accessed daily, retrieval fee applies
  • EFS One Zone — lower cost, single AZ, good for dev/test
  • EFS One Zone-IA — cheapest option, single AZ, infrequent access

Use EFS Lifecycle Management to automatically move files not accessed for a set period (7, 14, 30, 60, 90 days) to the IA storage class.

EFS vs EBS vs S3

Feature          EFS                  EBS                  S3
──────────────────────────────────────────────────────────────────────
Type             Shared file system   Block storage        Object storage
Protocol         NFS                  iSCSI                HTTP/S3 API
Concurrent access  Multiple EC2 ✅    Single EC2 only ❌   Multiple clients ✅
Scaling          Automatic            Manual (resize)      Unlimited
Use case         Shared CMS, web,     OS, DBs, apps        Backups, media, web
                 home dirs
AZ               Multi-AZ             Single AZ            Regional (multi-AZ)
Linux only       ✅ Yes               ❌ No (Win/Linux)     ✅ No (HTTP-based)

Security Considerations

  • Control access using Security Groups on the EFS mount target
  • Use IAM policies and EFS access points for fine-grained file system access
  • Enable KMS encryption at rest — must be configured at creation
  • Enable TLS encryption in transit when mounting
  • Use VPC to isolate EFS — it is not accessible from the public internet

AWS Solutions Architect Exam Notes

  • EFS = shared file system for multiple EC2 instances across multiple AZs
  • EBS = block storage attached to a single EC2 instance in one AZ
  • S3 = object storage accessible via API, not mountable as a file system
  • EFS is Linux only — uses NFS protocol; not compatible with Windows
  • EFS automatically scales — no need to provision capacity
  • Use EFS for: shared content repositories, web serving, home directories, CMS
  • EFS One Zone is cheaper but loses data if the AZ fails — use for non-critical data
  • EFS is more expensive than EBS per GB — use only when shared access is needed

Key Takeaways

  • EFS is a managed NFS file system shared across multiple EC2 instances and AZs
  • Scales automatically — no capacity planning needed
  • Use EFS when multiple instances need simultaneous read/write access to the same files
  • Use EBS for single-instance block storage; S3 for object storage via API
  • Enable lifecycle management to automatically reduce storage costs
