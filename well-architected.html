What is the Well-Architected Framework?

The AWS Well-Architected Framework provides a consistent approach for evaluating architectures and implementing designs that will scale over time. It is built on six pillars that represent best practices for building reliable, secure, efficient, cost-effective and sustainable systems in the cloud.

The AWS Well-Architected Tool is a free service in the console that lets you review your architecture against best practices and get improvement recommendations.

Pillar 1: Operational Excellence

The ability to support development and run workloads effectively, gain insight into operations, and continuously improve processes.

Key Principles

  • Perform operations as code (Infrastructure as Code — CloudFormation, CDK)
  • Make frequent, small, reversible changes
  • Refine operations procedures frequently
  • Anticipate failure — game days, chaos engineering
  • Learn from all operational failures

Key AWS Services

  • AWS CloudFormation / CDK — IaC
  • AWS Config — track resource configuration changes
  • AWS CloudWatch — monitor operations
  • AWS Systems Manager — operational management
  • AWS X-Ray — distributed tracing

Pillar 2: Security

The ability to protect data, systems and assets while delivering business value through risk assessments and mitigation strategies.

Key Principles

  • Implement a strong identity foundation — least privilege IAM
  • Enable traceability — CloudTrail, CloudWatch Logs
  • Apply security at all layers — network, compute, data
  • Automate security best practices
  • Protect data in transit and at rest — KMS, TLS
  • Keep people away from data — reduce manual access to production
  • Prepare for security events — incident response runbooks

Key AWS Services

  • IAM, AWS Organizations, SCP
  • AWS KMS, Secrets Manager
  • CloudTrail, GuardDuty, Security Hub
  • AWS WAF, Shield, Macie, Inspector

Pillar 3: Reliability

The ability of a workload to perform its intended function correctly and consistently when expected, and to recover quickly from failures.

Key Principles

  • Automatically recover from failure
  • Test recovery procedures — know your RPO and RTO
  • Scale horizontally to increase aggregate workload availability
  • Stop guessing capacity — use Auto Scaling
  • Manage change through automation

Key AWS Services

  • Multi-AZ deployments — RDS Multi-AZ, ELB across AZs
  • Auto Scaling — maintain desired capacity
  • Route 53 — health checks and failover routing
  • S3, RDS, DynamoDB — built-in replication and durability
  • AWS Backup — centralised backup management

Pillar 4: Performance Efficiency

The ability to use computing resources efficiently to meet requirements and maintain efficiency as demand changes and technologies evolve.

Key Principles

  • Democratize advanced technologies — use managed services
  • Go global in minutes — deploy in multiple regions
  • Use serverless architectures — Lambda, Fargate
  • Experiment more often
  • Consider mechanical sympathy — choose the right tool for the job

Key AWS Services

  • Auto Scaling, ELB — right-size compute
  • CloudFront, Global Accelerator — reduce latency globally
  • ElastiCache — in-memory caching
  • Lambda, Fargate — serverless compute
  • Amazon Aurora — high-performance managed database

Pillar 5: Cost Optimization

The ability to run systems to deliver business value at the lowest price point.

Key Principles

  • Implement cloud financial management
  • Adopt a consumption model — pay only for what you use
  • Measure overall efficiency
  • Stop spending money on undifferentiated heavy lifting — use managed services
  • Analyse and attribute expenditure — use Cost Allocation Tags

Key AWS Services

  • Reserved Instances / Savings Plans — commit for discounts
  • Spot Instances — up to 90% savings for flexible workloads
  • S3 Intelligent-Tiering — automatic cost-optimised storage
  • AWS Cost Explorer, Budgets, Trusted Advisor
  • Auto Scaling — scale in during low demand

Pillar 6: Sustainability

The ability to minimise the environmental impact of running cloud workloads. Added in 2021 as the sixth pillar.

Key Principles

  • Understand your impact — measure carbon footprint
  • Establish sustainability goals
  • Maximise utilisation — right-size and consolidate
  • Use more efficient hardware — AWS Graviton processors
  • Use managed services — AWS manages infrastructure efficiency
  • Reduce downstream impact — optimise code and assets

AWS Solutions Architect Exam Notes

  • There are 6 pillars: Operational Excellence, Security, Reliability, Performance Efficiency, Cost Optimization, Sustainability
  • Know the key design principles and AWS services for each pillar
  • Reliability = Multi-AZ, Auto Scaling, health checks, backup/restore
  • Security = least privilege IAM, encryption, audit trails, defence in depth
  • Cost Optimization = Reserved/Spot Instances, right-sizing, Auto Scaling, S3 tiers
  • Performance Efficiency = serverless, caching, CDN, right instance types
  • Operational Excellence = IaC, small reversible changes, monitor and improve
  • Sustainability = reduce footprint, right-size, use efficient hardware
  • The AWS Well-Architected Tool in the console helps review workloads against these pillars
  • Exam often asks: "which pillar does X best practice belong to?" — know the principles of each
