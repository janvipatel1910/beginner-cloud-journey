🪣 Amazon Simple Storage Service (S3)

Storage Service Core AWS Service AWS Solutions Architect Associate High Exam Importance
Overview Features Core Concepts Storage Classes Security Architecture Use Cases Exam Notes Summary

Service Overview

Amazon Simple Storage Service (S3) is an object storage service that offers industry-leading scalability, data availability, security and performance.

S3 provides 99.999999999% (11 nines) of durability and can store virtually unlimited amounts of data. Objects can range in size from 0 bytes to 5 terabytes.

Key Features

  • 99.999999999% (11 nines) durability for stored objects
  • Versioning — keep multiple versions of an object
  • Lifecycle policies — automatically transition or delete objects
  • Static website hosting — serve HTML/CSS/JS directly from S3
  • Event notifications — trigger Lambda functions on object changes
  • Cross-Region Replication (CRR) for disaster recovery
  • Transfer Acceleration using CloudFront edge locations

Core Concepts

Buckets

A bucket is a container for objects stored in S3. Bucket names must be globally unique across all AWS accounts. Buckets are created in a specific AWS Region.

Objects

An object is any file stored in S3 — images, videos, logs, backups, HTML files. Each object consists of the data (file) and metadata (key, size, type).

Keys

A key is the unique identifier (file path) for an object within a bucket. Example: images/profile/photo.jpg

Versioning

When versioning is enabled on a bucket, S3 keeps all versions of every object. This protects against accidental deletion and overwrites.

Storage Classes

  • S3 Standard — frequently accessed data, highest availability
  • S3 Standard-IA — infrequently accessed, lower cost, retrieval fee
  • S3 One Zone-IA — infrequently accessed, stored in single AZ, cheapest IA option
  • S3 Glacier Instant Retrieval — archive data, millisecond retrieval
  • S3 Glacier Flexible Retrieval — archive data, retrieval in minutes to hours
  • S3 Glacier Deep Archive — lowest cost, retrieval in 12+ hours
  • S3 Intelligent-Tiering — automatically moves objects between tiers based on access patterns

Security Considerations

  • Block Public Access — enabled by default, prevents unintended public exposure
  • Bucket Policies — resource-based JSON policies controlling bucket-level access
  • Access Control Lists (ACLs) — legacy per-object access control
  • Server-Side Encryption (SSE) — encrypt data at rest using SSE-S3, SSE-KMS or SSE-C
  • Presigned URLs — temporary URLs granting time-limited access to private objects
  • MFA Delete — require MFA to delete objects when versioning is enabled

Architecture Overview

AWS Region
    │
    └── S3 Bucket (globally unique name)
            │
            ├── Object: images/logo.png
            ├── Object: videos/intro.mp4
            └── Object: docs/readme.txt

Real-World Use Cases

  • Static website hosting — serve a complete website directly from S3
  • Backup and disaster recovery — store encrypted backups reliably
  • Data lake — centralize structured and unstructured data for analytics
  • Media storage — host images, videos and audio files for applications
  • Log storage — archive application and infrastructure logs
  • Software distribution — deliver large files globally via S3 + CloudFront

AWS Solutions Architect Exam Notes

  • S3 bucket names must be globally unique
  • S3 is a regional service but bucket names are globally unique
  • S3 Standard provides 99.99% availability and 11 nines durability
  • Use Lifecycle Policies to move objects to cheaper storage classes over time
  • Versioning must be enabled before MFA Delete can be configured
  • Block Public Access should be enabled unless serving a public static website
  • Use S3 Transfer Acceleration for faster uploads from remote locations
  • Cross-Region Replication requires versioning enabled on both source and destination
  • S3 is an object store, not a file system — use EFS for file system needs

Key Takeaways

  • S3 is a highly durable, scalable object storage service
  • Bucket names are globally unique; objects are stored with keys
  • Storage classes let you optimize cost based on access frequency
  • Enable versioning to protect against accidental deletion
  • Block Public Access by default; use bucket policies and SSE for security
