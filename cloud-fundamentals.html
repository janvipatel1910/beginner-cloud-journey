Cloud Fundamentals is the foundation of AWS and one of the first topics for the AWS Solutions Architect Associate certification.

What is Cloud Computing?

Cloud Computing is the delivery of computing services — including servers, storage, databases, networking, software, analytics and intelligence — over the internet, enabling faster innovation, flexible resources and economies of scale.

Benefits of Cloud Computing

  • Pay-as-you-go pricing — only pay for what you use
  • Scalability — scale up or down based on demand
  • High Availability — redundancy across multiple locations
  • Global Reach — deploy applications worldwide in minutes
  • Improved Security — AWS manages physical infrastructure security
  • Elasticity — automatically adjust resources to match workload
  • Speed and Agility — provision resources in minutes

Cloud Service Models

Cloud services are commonly categorized into three models: Infrastructure as a Service (IaaS), Platform as a Service (PaaS), and Software as a Service (SaaS).

  • IaaS – Provides virtualized computing resources. Example: Amazon EC2
  • PaaS – Provides a platform for developers to build applications. Example: AWS Elastic Beanstalk
  • SaaS – Delivers software over the internet. Example: Gmail, Microsoft 365

Cloud Deployment Models

  • Public Cloud – Resources owned and operated by a cloud provider (e.g. AWS)
  • Private Cloud – Resources used exclusively by one organization
  • Hybrid Cloud – Combination of public and private cloud environments

AWS Global Infrastructure

AWS provides infrastructure across the world through Regions, Availability Zones and Edge Locations.

Region

A Region is a geographical area containing multiple Availability Zones. Example: eu-west-2 (London), us-east-1 (N. Virginia).

Availability Zone (AZ)

An Availability Zone is one or more data centers within a Region, each with independent power, cooling and networking. Example: London Region → AZ-A, AZ-B, AZ-C.

Edge Location

Edge Locations are used by CloudFront to cache content closer to users, reducing latency and improving performance. There are more Edge Locations than Regions.

AWS Shared Responsibility Model

AWS follows a Shared Responsibility Model — security responsibilities are divided between AWS and the customer.

  • AWS is responsible for security of the cloud — physical infrastructure, hardware, networking, managed services
  • Customers are responsible for security in the cloud — IAM permissions, operating systems, applications, encryption and data

Real-World Use Cases

  • Netflix streaming services hosted on AWS
  • Online banking applications with high availability requirements
  • E-commerce platforms such as Amazon itself
  • Global content delivery networks for media companies
  • Startups scaling from zero to millions of users without upfront hardware

AWS Solutions Architect Exam Notes

  • Regions contain multiple Availability Zones (minimum of 2, typically 3)
  • Availability Zones consist of one or more data centers
  • CloudFront uses Edge Locations for content caching — there are more Edge Locations than Regions
  • AWS follows the Shared Responsibility Model — AWS secures the infrastructure, customers secure their data
  • Cloud computing provides scalability, elasticity and pay-as-you-go pricing
  • IaaS, PaaS and SaaS are the three cloud service models
