🔐 AWS Identity and Access Management (IAM)

IAM is one of the most important AWS services and is heavily tested in the AWS Solutions Architect Associate exam.

📚 Lesson Information

⏱️ Estimated Study Time: 20 Minutes

🎯 Difficulty Level: Beginner

⭐ Exam Importance: Very High

What is IAM?

AWS Identity and Access Management (IAM) allows you to securely control access to AWS services and resources.

IAM helps you decide:

  • Who can access AWS?
  • What resources can they access?
  • What actions can they perform?

Why IAM Matters

  • 🔒 Improve Security
  • 👤 Manage Users
  • 🎭 Create Temporary Access
  • 📜 Control Permissions
  • 🛡️ Apply Least Privilege Access

IAM Components

👤 Users

An IAM User represents a person or application that needs access to AWS.

👥 Groups

Groups are collections of users. Permissions can be assigned to a group instead of assigning permissions individually.

🎭 Roles

Roles provide temporary permissions and are commonly used by AWS services such as EC2 and Lambda.

📜 Policies

Policies are JSON documents that define permissions in AWS.

🔐 IAM Structure

``` 
AWS Account
     │
     ├── Users
     │
     ├── Groups
     │
     ├── Roles
     │
     └── Policies
```

Least Privilege Principle

Always give users only the permissions they need to perform their tasks. Nothing more.

This reduces security risks and follows AWS best practices.

Real World Example

A Cloud Engineer may need access to EC2 and S3.

They should not automatically receive permissions for billing, security administration or deleting production resources.

🎯 Exam Tips

  • Users represent people or applications.
  • Groups contain multiple users.
  • Roles provide temporary access.
  • Policies define permissions.
  • IAM is a Global AWS Service.

📝 Quick Quiz

Which IAM component defines permissions?

A. User
B. Policy
C. Group
D. Role

Answer: B. Policy
