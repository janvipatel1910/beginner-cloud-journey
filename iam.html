AWS Identity and Access Management (IAM)

Service Overview

AWS Identity and Access Management (IAM) is a global AWS service used to securely control access to AWS resources.

IAM enables organizations to define who can access AWS resources, which resources they can access, and what actions they are authorized to perform.

Core Concepts

Authentication

Authentication verifies the identity of a user, application, or service attempting to access AWS resources.

  • Username and Password
  • Access Keys
  • Multi-Factor Authentication (MFA)

Authorization

Authorization determines which AWS resources an authenticated identity can access and what actions can be performed.

IAM Components

IAM Users

An IAM User represents an individual identity within an AWS account.

IAM Groups

IAM Groups provide a mechanism for managing permissions across multiple users.

IAM Roles

IAM Roles provide temporary security credentials.

IAM Policies

Policies are JSON documents that define permissions.

Architecture Overview

AWS Account
│
├── IAM Users
│
├── IAM Groups
│
├── IAM Roles
│
└── IAM Policies

Security Best Practices

  • Apply the Principle of Least Privilege
  • Enable Multi-Factor Authentication (MFA)
  • Avoid using the Root Account
  • Rotate credentials regularly
  • Use IAM Roles instead of Access Keys

AWS Solutions Architect Exam Notes

  • IAM is a Global Service
  • Policies define permissions
  • Groups cannot contain other groups
  • IAM Roles provide temporary credentials
  • Follow the Principle of Least Privilege

Key Takeaways

  • IAM controls authentication and authorization.
  • Users represent identities.
  • Groups simplify permission management.
  • Roles provide temporary access.
  • Policies define permissions.
