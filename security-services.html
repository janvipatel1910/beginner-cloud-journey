Amazon GuardDuty

GuardDuty is a threat detection service that continuously monitors your AWS accounts and workloads for malicious activity and unauthorised behaviour. It uses machine learning, anomaly detection and threat intelligence feeds.

What GuardDuty Analyses

  • CloudTrail logs — unusual API calls, suspicious console logins
  • VPC Flow Logs — unusual network traffic patterns
  • DNS logs — communication with known malicious domains
  • EKS audit logs — suspicious Kubernetes activity

Key Points

  • Enable with one click — no agents or software to install
  • Generates findings with severity levels: Low, Medium, High
  • Findings sent to EventBridge → Lambda / SNS for automated remediation
  • Supports multi-account via AWS Organizations
  • 30-day free trial

Amazon Macie

Macie is a data security service that uses machine learning to automatically discover, classify and protect sensitive data in Amazon S3. It identifies Personally Identifiable Information (PII), financial data, credentials and other sensitive content.

Key Points

  • Scans S3 buckets for sensitive data (PII, credit cards, AWS credentials, etc.)
  • Alerts on buckets that are publicly accessible, unencrypted or shared externally
  • Findings sent to EventBridge for automated remediation
  • Essential for GDPR, HIPAA and PCI-DSS compliance
  • Enable per-region — must be enabled in each region where you have S3 data

Amazon Inspector

Inspector is a vulnerability assessment service that automatically assesses applications for security vulnerabilities and deviations from best practices.

What Inspector Scans

  • EC2 instances — OS vulnerabilities, network accessibility
  • Container images in ECR — scan for CVEs before deployment
  • Lambda functions — software vulnerabilities in function code and dependencies

Key Points

  • Automatic and continuous scanning — not a one-time scan
  • Produces a risk score for each finding based on severity
  • Integrates with AWS Security Hub for centralised findings
  • No need to install agents for EC2 — uses SSM Agent

AWS WAF (Web Application Firewall)

AWS WAF protects your web applications from common exploits such as SQL injection and cross-site scripting (XSS). It lets you create rules that control which web requests reach your application.

Key Points

  • Deployed in front of: CloudFront, ALB, API Gateway, AppSync
  • Rules can allow, block or count requests based on IP, headers, body, URI, country
  • Managed Rule Groups — pre-built rules from AWS and security vendors (OWASP Top 10)
  • Rate-based rules — block IPs sending too many requests (DDoS mitigation)
  • Real-time visibility into web traffic via CloudWatch metrics

AWS Shield

AWS Shield protects AWS applications against Distributed Denial of Service (DDoS) attacks.

Shield Standard

  • Automatically included for all AWS customers at no extra cost
  • Protects against common, most frequently occurring network and transport layer DDoS attacks
  • Active on CloudFront, Route 53, ELB automatically

Shield Advanced

  • $3,000/month per organisation
  • Protection against larger and more sophisticated DDoS attacks
  • 24/7 access to AWS DDoS Response Team (DRT)
  • Financial protection — AWS credits back costs incurred due to DDoS scaling
  • Protects: EC2, ELB, CloudFront, Route 53, Global Accelerator
  • Real-time attack visibility and notifications

AWS Config

AWS Config is a service that assesses, audits and evaluates the configuration of your AWS resources over time. It answers: "What did my infrastructure look like at any point in time?" and "Is my infrastructure compliant with my rules?"

Key Points

  • Records configuration changes to AWS resources continuously
  • Config Rules — evaluate whether resources comply with your policies
  • Example rules: "All EBS volumes must be encrypted", "All S3 buckets must have versioning"
  • Remediation actions — automatically fix non-compliant resources using SSM Automation
  • Per-region service — but can aggregate findings across regions and accounts
  • Config ≠ CloudTrail — Config tracks what the configuration is; CloudTrail tracks who changed it

AWS Solutions Architect Exam Notes

  • GuardDuty = threat detection, ML-based, analyses CloudTrail + VPC Flow Logs + DNS
  • Macie = sensitive data in S3 (PII, credit cards, credentials)
  • Inspector = vulnerability scanning for EC2, ECR, Lambda
  • WAF = protect against SQL injection, XSS, rate limiting — deploys in front of CloudFront/ALB
  • Shield Standard = free, automatic DDoS protection on CloudFront and Route 53
  • Shield Advanced = $3,000/month, protects EC2 and ELB, 24/7 DRT support
  • Config = resource compliance and configuration history — "was port 22 open last Tuesday?"
  • GuardDuty findings → EventBridge → Lambda = automated threat remediation pattern
  • WAF + Shield Advanced = comprehensive DDoS and application-layer protection
