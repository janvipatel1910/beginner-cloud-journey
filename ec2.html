🖥️ Amazon Elastic Compute Cloud (EC2)

Compute Service Core AWS Service AWS Solutions Architect Associate High Exam Importance
Overview Features Instance Types Pricing Storage Security Architecture Use Cases Exam Notes Summary

Service Overview

Amazon Elastic Compute Cloud (EC2) is a web service that provides secure, resizable virtual servers (instances) in the AWS Cloud.

EC2 allows organizations to deploy applications without purchasing and managing physical hardware — you pay only for the compute capacity you actually use.

Key Features

  • On-demand virtual servers — launch in minutes
  • Auto Scaling support — scale in or out based on demand
  • Elastic Load Balancer integration for high availability
  • Multiple operating systems — Amazon Linux, Ubuntu, Windows Server
  • Flexible pricing options — On-Demand, Reserved, Spot
  • Elastic IP Addresses for static public IPs

EC2 Instance Types

AWS provides different instance families optimized for different workloads.

  • General Purpose (t3, t4g, m5) — balanced compute, memory, networking
  • Compute Optimized (c5, c6) — high-performance processing
  • Memory Optimized (r5, x1) — large in-memory datasets
  • Storage Optimized (i3, d2) — high-speed local storage
  • Accelerated Computing (p3, g4) — GPU workloads and machine learning

Pricing Models

  • On-Demand — pay per hour or second, no commitment
  • Reserved Instances — 1 or 3 year commitment, up to 75% savings
  • Spot Instances — bid on spare capacity, up to 90% savings, can be interrupted
  • Savings Plans — flexible commitment to usage, significant discounts

Storage Options

  • Amazon EBS — Elastic Block Store, persistent network-attached storage
  • Instance Store — temporary storage on the physical host, lost on stop/terminate
  • Amazon EFS — Elastic File System, shared file storage across multiple instances

Security

Security Groups act as virtual firewalls that control inbound and outbound traffic to EC2 instances. They are stateful — if you allow inbound traffic, the response is automatically allowed.

IAM Roles should be attached to EC2 instances to grant access to other AWS services — never store AWS Access Keys inside an EC2 instance directly.

Real-World Use Cases

  • Website and application hosting
  • Development and testing environments
  • Machine learning and batch processing workloads
  • Enterprise application migration from on-premises
  • Game server hosting with auto scaling

AWS Solutions Architect Exam Notes

  • EC2 is a regional AWS service
  • Security Groups are stateful — return traffic is automatically allowed
  • EBS provides persistent storage that survives instance stop/start
  • Instance Store is ephemeral — data is lost when the instance stops
  • IAM Roles are preferred over access keys for EC2-to-AWS service access
  • Reserved Instances save up to 75% vs On-Demand for predictable workloads
  • Spot Instances are cheapest but can be interrupted by AWS with 2-minute notice

Key Takeaways

  • EC2 provides scalable virtual servers in the AWS Cloud
  • Security Groups control network traffic — stateful, instance-level firewall
  • Choose the right instance type for the workload (compute, memory, storage, GPU)
  • EBS persists data; Instance Store does not
  • Use IAM Roles — never hard-code credentials in instances
