Service Overview

Amazon Elastic Compute Cloud (EC2) is a web service that provides secure and resizable virtual servers in the AWS Cloud.

EC2 allows organizations to deploy applications without purchasing and managing physical hardware.

Key Features

  • On-demand virtual servers
  • Auto Scaling support
  • Elastic Load Balancer integration
  • Multiple operating systems
  • Flexible pricing options

EC2 Instance Types

AWS provides different instance families for different workloads.

  • General Purpose (t3, t4g, m5)
  • Compute Optimized (c5, c6)
  • Memory Optimized (r5, x1)
  • Storage Optimized (i3, d2)

Storage Options

  • Amazon EBS (Persistent Storage)
  • Instance Store (Temporary Storage)
  • Amazon EFS (Shared File Storage)

Security

Security Groups act as virtual firewalls that control inbound and outbound traffic.

IAM Roles should be used instead of storing AWS Access Keys inside EC2 instances.

AWS Solutions Architect Exam Notes

  • EC2 is a regional AWS service.
  • Security Groups are stateful.
  • EBS provides persistent storage.
  • IAM Roles are preferred over access keys.
  • Auto Scaling improves availability.

Key Takeaways

  • EC2 provides virtual servers in AWS.
  • Security Groups control network traffic.
  • EBS stores data persistently.
  • IAM Roles improve security.
  • EC2 is a core AWS service.
