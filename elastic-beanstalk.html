🌱 AWS Elastic Beanstalk

PaaS Application Deployment AWS Solutions Architect Associate Medium Exam Importance
Overview Features Core Concepts Deployment Policies Security Exam Notes Summary

Service Overview

AWS Elastic Beanstalk is a Platform as a Service (PaaS) that makes it easy to deploy and run applications in multiple languages. You upload your code and Beanstalk automatically handles capacity provisioning, load balancing, auto scaling, health monitoring and deployment.

You retain full control over the AWS resources powering the application and can access the underlying resources at any time. Elastic Beanstalk itself is free — you only pay for the underlying resources it provisions (EC2, RDS, ELB, etc.).

Key Features

  • Supports: Node.js, Python, Ruby, PHP, Java, .NET, Go and Docker
  • Automatically provisions EC2, ELB, Auto Scaling, RDS and CloudWatch
  • Managed platform updates — Beanstalk can keep the runtime patched
  • Multiple deployment policies for zero-downtime deployments
  • Environment cloning — duplicate environments for staging/production
  • Integrated with CodePipeline for CI/CD
  • Free to use — pay only for resources consumed

Core Concepts

Application

A logical collection of Beanstalk components — environments, versions and configurations.

Application Version

A specific, labelled iteration of deployable code stored in S3.

Environment

A collection of AWS resources running an application version. Each environment runs only one application version at a time. You can have multiple environments (dev, staging, production) per application.

Environment Tiers

  • Web Server Tier — handles HTTP requests, runs EC2 with ELB and Auto Scaling
  • Worker Tier — processes background tasks from an SQS queue. No ELB — pulls jobs from SQS.

Deployment Policies

  • All at once — deploys to all instances simultaneously. Fastest but causes downtime.
  • Rolling — deploys in batches. Reduced capacity during deployment. No downtime.
  • Rolling with additional batch — launches extra instances first, then rolls. Maintains full capacity. No downtime.
  • Immutable — launches a completely new set of instances. Safest. No downtime. Easy rollback.
  • Blue/Green — deploys to a separate environment, then swaps CNAME. Zero downtime, instant rollback.
  • Traffic splitting — sends a percentage of traffic to the new version (canary deployment).

Security Considerations

  • Assign an IAM instance profile to EC2 instances — grants permissions without hard-coded credentials
  • Use Secrets Manager for environment variables containing credentials
  • Place RDS outside the Beanstalk environment — so it persists when the environment is terminated
  • Enable HTTPS on the ELB listener, not at the EC2 level

AWS Solutions Architect Exam Notes

  • Beanstalk is free — pay only for EC2, RDS, ELB etc. it provisions
  • Beanstalk is PaaS — developer uploads code, AWS handles infrastructure
  • Immutable deployment = safest, no capacity reduction, easy rollback
  • Blue/Green = swap CNAME between two environments — instant rollback
  • Worker tier + SQS = decouple long-running background tasks from web tier
  • Beanstalk environments can be cloned for quick staging setup
  • Never put RDS inside the Beanstalk environment — it gets deleted when the env is terminated
  • Beanstalk is not for microservices — use ECS/Fargate for containerised microservices

Key Takeaways

  • Elastic Beanstalk = PaaS for deploying web apps without managing infrastructure
  • Supports multiple languages and Docker
  • Choose deployment policy based on downtime tolerance and rollback needs
  • Use Worker tier + SQS for background processing
  • Always keep RDS outside the Beanstalk environment for data persistence
