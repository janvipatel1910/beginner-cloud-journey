🔄 AWS Step Functions

Serverless Orchestration Workflows AWS Solutions Architect Associate Medium Exam Importance
Overview Features Core Concepts Workflow Types Architecture Exam Notes Summary

Service Overview

AWS Step Functions is a serverless orchestration service that lets you coordinate multiple AWS services into serverless workflows. It allows you to build multi-step applications where the output of one step becomes the input of the next.

Step Functions visualise workflows as state machines — making complex distributed applications easier to build, run and debug.

Key Features

  • Visual workflow designer — build and visualise state machines
  • Built-in error handling, retries and timeouts
  • Supports parallel execution and conditional branching
  • Integrates with 200+ AWS services natively
  • Audit trail — view complete execution history
  • Two workflow types: Standard and Express

Core Concepts

State Machine

A state machine defines your workflow as a series of states using Amazon States Language (JSON). Each state performs work, makes decisions, or waits.

State Types

  • Task — calls an AWS service (Lambda, ECS, DynamoDB, SQS, etc.)
  • Choice — branches workflow based on a condition
  • Parallel — executes multiple branches simultaneously
  • Wait — pauses execution for a specified time
  • Pass — passes input directly to output (useful for testing)
  • Succeed / Fail — terminates execution successfully or with an error
  • Map — runs the same steps for each element in an array (like a for-each loop)

Workflow Types

Standard Workflows

  • For long-running workflows (up to 1 year)
  • At-most-once execution — each step runs once unless retried on failure
  • Full execution history visible in console
  • $0.025 per 1,000 state transitions

Express Workflows

  • For high-volume, short-duration workflows (up to 5 minutes)
  • At-least-once execution — steps may run more than once
  • Execution history via CloudWatch Logs only
  • Priced per execution and duration — cheaper for high volume

AWS Solutions Architect Exam Notes

  • Step Functions = orchestration of multiple Lambda functions or AWS services into a workflow
  • Use Step Functions when a workflow has multiple steps, branching logic, or error handling
  • Standard = long-running (up to 1 year), at-most-once, expensive at scale
  • Express = short-duration (up to 5 min), high-volume, cheaper
  • Step Functions replaces complex Lambda-chaining code with a visual state machine
  • Built-in retry and catch — no need to write retry logic in Lambda code
  • Use the Map state to process arrays in parallel — replaces fan-out Lambda patterns
  • Step Functions ≠ SQS — Step Functions orchestrates workflow steps; SQS decouples producers/consumers

Key Takeaways

  • Step Functions orchestrates multi-step workflows as visual state machines
  • Use when you need conditional branching, parallel execution, retries, or timeouts across services
  • Standard workflows for long-running; Express for high-volume short tasks
  • Eliminates complex error-handling code from Lambda functions
  • Integrates with 200+ AWS services — not just Lambda
