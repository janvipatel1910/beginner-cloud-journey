AWS Cost Management Tools

AWS Cost Explorer

  • Visualise and analyse your AWS costs and usage over time
  • Filter by service, region, account, tag, instance type
  • Forecast future spend based on historical usage
  • Identify Reserved Instance / Savings Plan purchase recommendations
  • Free to use

AWS Budgets

  • Set custom cost and usage budgets with alert notifications
  • Alert via email or SNS when actual or forecasted spend exceeds budget
  • Supports cost budgets, usage budgets, RI/Savings Plan utilisation budgets
  • First two budgets free; $0.02/day per budget thereafter

AWS Trusted Advisor

  • Provides real-time recommendations across five categories:
  • Cost Optimization — identify idle resources, underutilised instances, unused EIPs
  • Performance — identify over/under-provisioned resources
  • Security — identify open ports, missing MFA, exposed access keys
  • Fault Tolerance — identify lack of Multi-AZ, insufficient backups
  • Service Limits — warn when approaching service quotas
  • Basic checks: free for all accounts; Full checks: require Business or Enterprise Support

AWS Cost and Usage Report (CUR)

  • Most comprehensive cost and usage data available
  • Delivered as CSV files to S3
  • Analyse with Athena, QuickSight or third-party tools
  • Use for detailed billing analysis and chargeback/showback

Cost Allocation Tags

  • Tag AWS resources with key-value pairs (e.g. Department=Engineering, Project=Alpha)
  • Activate tags in Billing console to see costs broken down by tag
  • Essential for attributing costs to teams, projects or environments

EC2 Pricing Models

On-Demand

  • Pay per second/hour — no commitment
  • Highest cost but maximum flexibility
  • Use for: unpredictable workloads, short-term needs, development

Reserved Instances (RI)

  • 1 or 3 year commitment — up to 72% savings vs On-Demand
  • Standard RI — fixed instance family, OS, region; highest discount
  • Convertible RI — can change instance type/OS; smaller discount (~54%)
  • Payment options: All Upfront, Partial Upfront, No Upfront (No Upfront is cheapest upfront but lowest discount)
  • Can sell unused Standard RIs on the Reserved Instance Marketplace

Savings Plans

  • Flexible pricing model — commit to a consistent usage amount ($/hour) for 1 or 3 years
  • Compute Savings Plans — up to 66% savings, flexible across instance family, region, OS, and compute type (EC2, Lambda, Fargate)
  • EC2 Instance Savings Plans — up to 72%, but locked to a specific instance family in a region
  • Simpler than RIs — no need to specify instance type upfront for Compute plans

Spot Instances

  • Use unused EC2 capacity — up to 90% savings vs On-Demand
  • AWS can reclaim with 2-minute notice — instances are not guaranteed
  • Use for: fault-tolerant batch jobs, data analysis, rendering, CI/CD
  • Never use for: databases, critical web servers, stateful applications
  • Spot Fleets — mix of Spot and On-Demand instances to meet target capacity

Dedicated Hosts

  • Physical server dedicated to your use — no sharing with other customers
  • Required for compliance and software licensing (bring-your-own-licence — BYOL)
  • Most expensive EC2 option

Storage Cost Optimization

  • S3 Intelligent-Tiering — automatically moves objects between access tiers; no retrieval fees
  • S3 Lifecycle Policies — automatically transition objects to Standard-IA, then Glacier after set periods
  • S3 Glacier / Deep Archive — for long-term archival data (regulatory, compliance)
  • EBS — delete unused snapshots; right-size volumes; use gp3 over gp2 (cheaper per GB)
  • EFS Lifecycle Management — auto-move infrequent files to EFS-IA storage class

Architectural Best Practices

  • Right-sizing — use CloudWatch metrics to identify underutilised instances and downsize
  • Auto Scaling — scale in during off-peak hours; don't pay for idle capacity
  • Serverless — Lambda and Fargate charge only for actual use — no idle costs
  • Managed services — RDS, ElastiCache, SQS reduce operational overhead and infrastructure waste
  • Stop/start non-production environments — use Lambda + EventBridge to shut down dev/test EC2 overnight
  • Delete unattached EBS volumes and unused EIPs — both incur charges when not in use

AWS Solutions Architect Exam Notes

  • Spot Instances = up to 90% savings, interruptible — use for batch/fault-tolerant work
  • Reserved Instances / Savings Plans = up to 72% savings for predictable, steady-state workloads
  • On-Demand = no commitment, most expensive — use for unpredictable workloads
  • Trusted Advisor — full checks require Business or Enterprise Support
  • Cost Explorer — free, visualise costs, forecasts, RI recommendations
  • AWS Budgets — set alerts when spend exceeds threshold
  • Unattached EBS volumes and unused Elastic IPs charge money — delete them
  • Cost Allocation Tags must be activated in Billing console before they appear in reports
  • Convertible RIs can change instance type; Standard RIs cannot — but Standard has higher discount
  • gp3 EBS is cheaper and faster than gp2 — always prefer gp3 for new volumes

Key Takeaways

  • Use Cost Explorer, Budgets and Trusted Advisor to monitor and optimise spend
  • Match EC2 pricing model to workload: Spot for batch, Reserved/Savings Plans for steady-state, On-Demand for variable
  • Use S3 Lifecycle Policies and Intelligent-Tiering to reduce storage costs automatically
  • Right-size instances, delete unused resources, and use Auto Scaling to eliminate waste
  • Serverless and managed services reduce both cost and operational overhead
