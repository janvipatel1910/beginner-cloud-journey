🔌 AWS Storage Gateway & Snow Family

Hybrid Storage Data Migration AWS Solutions Architect Associate Medium Exam Importance
Storage Gateway Snow Family DataSync Exam Notes Summary

AWS Storage Gateway

AWS Storage Gateway is a hybrid cloud storage service that gives your on-premises applications access to virtually unlimited cloud storage. It bridges on-premises environments with AWS cloud storage.

Gateway Types

S3 File Gateway

  • On-premises applications access S3 objects using NFS or SMB protocols
  • Files are stored as objects in S3 — accessed like a local file share
  • Use for: file backups, content repositories, tiered storage

FSx File Gateway

  • Low-latency access to Amazon FSx for Windows File Server from on-premises
  • Use for: Windows home directories, departmental shares

Volume Gateway

  • Presents iSCSI block storage volumes to on-premises applications
  • Cached volumes — primary data in S3, frequently accessed data cached on-premises
  • Stored volumes — primary data on-premises, async backup to S3 as EBS snapshots

Tape Gateway

  • Virtual tape library (VTL) interface to replace physical tape backups
  • Works with existing backup software (Veeam, Backup Exec, etc.)
  • Virtual tapes stored in S3 and archived to S3 Glacier for long-term retention

AWS Snow Family

The Snow Family provides physical devices to collect and process data at the edge and migrate large amounts of data into and out of AWS when network transfer would be too slow or expensive.

Snowcone

  • Smallest device — 8TB HDD or 14TB SSD
  • Rugged, portable — fits in a backpack
  • Use for: edge computing in remote or harsh environments
  • Can send data to AWS via internet (DataSync) or by mailing the device back

Snowball Edge

  • Storage Optimised — 80TB usable storage, 40 vCPU
  • Compute Optimised — 28TB NVMe storage, 52 vCPU + optional GPU
  • Use for: large data migrations, edge computing, disconnected environments
  • Run EC2 instances and Lambda functions directly on the device at the edge

Snowmobile

  • Massive data transfer — up to 100 petabytes per Snowmobile
  • A 45-foot shipping container pulled by a semi-truck
  • Use for: migrating entire data centres to AWS
  • AWS security personnel escort the truck and GPS tracking is included

Choosing the Right Snow Device

Data Size          Device
────────────────────────────────
< 10TB             Use internet transfer
10TB – 10PB        Snowball Edge (one or multiple)
> 10PB             Snowmobile
Remote/harsh env   Snowcone

AWS DataSync

DataSync is an online data transfer service that automates moving data between on-premises storage and AWS storage services (S3, EFS, FSx). It transfers data over the internet or Direct Connect, using a lightweight agent installed on-premises.

  • Up to 10x faster than open-source tools
  • Handles scheduling, bandwidth throttling, encryption and data integrity checks
  • Use for: ongoing sync between on-premises and AWS, data migration, replication
  • DataSync = online transfer via network; Snow Family = offline transfer via physical device

AWS Solutions Architect Exam Notes

  • S3 File Gateway = on-premises apps access S3 via NFS/SMB — for file-based cloud tiering
  • Volume Gateway = iSCSI block storage backed by S3 — for on-premises block storage with cloud backup
  • Tape Gateway = replace physical tape library with virtual tapes in S3/Glacier
  • Snowball Edge = physical device for migrating TBs–PBs to AWS offline
  • Snowmobile = shipping container for petabyte-scale migrations (>10PB)
  • DataSync = online data transfer between on-premises and AWS — not for offline/large migrations
  • If the question mentions limited bandwidth or large data → Snow Family
  • If the question mentions ongoing sync or scheduled transfers → DataSync
  • Snow devices can run EC2 instances at the edge — useful for remote/disconnected processing

Key Takeaways

  • Storage Gateway bridges on-premises storage with AWS cloud storage
  • S3 File Gateway, Volume Gateway and Tape Gateway cover files, blocks and backups
  • Snow Family physically ships data when network transfer is too slow or too expensive
  • Use DataSync for automated, ongoing online transfers between on-premises and AWS
  • Choose based on data size, connectivity and urgency
