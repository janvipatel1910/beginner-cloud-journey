🚌 Amazon EventBridge

Event Bus Serverless AWS Solutions Architect Associate High Exam Importance
Overview Features Core Concepts Security Architecture Use Cases Exam Notes Summary

Service Overview

Amazon EventBridge is a serverless event bus service that makes it easy to connect applications using events from AWS services, integrated SaaS applications and your own custom applications.

EventBridge was previously known as CloudWatch Events. It extends that service with support for SaaS integrations, a schema registry and an event archive.

Key Features

  • Receives events from AWS services, SaaS providers and custom applications
  • Routes events to targets based on rules (filter patterns)
  • Supports 20+ AWS services as targets: Lambda, SQS, SNS, Step Functions, ECS, and more
  • Scheduled rules — run tasks on a schedule using cron or rate expressions
  • Schema Registry — automatically discovers and stores event schemas
  • Event Archive and Replay — store events and replay them for debugging
  • Cross-account and cross-region event routing

Core Concepts

Event Bus

An event bus is a pipeline that receives events. There are three types:

  • Default event bus — receives events from AWS services automatically
  • Custom event bus — receives events from your own applications
  • Partner event bus — receives events from SaaS partners (Zendesk, Datadog, etc.)

Rules

A rule matches incoming events based on an event pattern and routes them to one or more targets. Rules can also be scheduled using cron or rate expressions.

Targets

Targets are the services that receive matched events. Common targets include: Lambda, SQS, SNS, Step Functions, ECS tasks, API Gateway, Kinesis, and more. A single rule can have up to 5 targets.

Event Pattern

An event pattern is a JSON filter that defines which events a rule matches. Events that match the pattern are sent to the rule's targets. Example: match all EC2 instance state change events where state = "stopped".

Security Considerations

  • Use IAM roles to grant EventBridge permission to invoke targets
  • Use resource-based policies on the event bus to control who can send events
  • For cross-account routing, grant explicit permission on the target account's event bus

        ▼

Real-World Use Cases

  • Trigger Lambda when an EC2 instance enters a "stopped" state
  • Run a database backup Lambda every night at midnight using a cron rule
  • React to SaaS events — e.g. trigger a workflow when a Zendesk ticket is created
  • Cross-account monitoring — route security events to a central security account
  • Replace polling with event-driven reactions to AWS service state changes

AWS Solutions Architect Exam Notes

  • EventBridge = modern replacement for CloudWatch Events — same underlying service, more features
  • Use EventBridge for scheduled tasks (cron/rate) and event-driven reactions to AWS service changes
  • Default event bus automatically receives all AWS service events
  • EventBridge is the best choice when you need to react to AWS service state changes (e.g. EC2 state, S3 events, CodePipeline stages)
  • Supports SaaS integration — SNS does not
  • Schema Registry helps developers discover event structures without reading documentation
  • Archive and Replay is useful for debugging and testing event-driven systems

Key Takeaways

  • EventBridge is a serverless event bus that routes events to AWS targets
  • Use rules to filter events by pattern and route to Lambda, SQS, Step Functions, etc.
  • Scheduled rules replace cron jobs on servers — fully serverless scheduling
  • Default bus receives AWS service events automatically — no configuration needed
  • EventBridge is the glue connecting event-driven AWS architectures
