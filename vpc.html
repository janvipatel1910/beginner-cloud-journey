🌐 Amazon Virtual Private Cloud (VPC)

Networking Service Core AWS Service AWS Solutions Architect Associate High Exam Importance
Overview Features Core Concepts Security Architecture Use Cases Exam Notes Summary

Service Overview

Amazon Virtual Private Cloud (VPC) lets you provision a logically isolated section of the AWS Cloud where you can launch AWS resources in a virtual network that you define.

You have full control over your virtual networking environment — including selection of your IP address range, creation of subnets, and configuration of route tables and network gateways.

Key Features

  • Custom IP address ranges using CIDR blocks (e.g. 10.0.0.0/16)
  • Public and private subnets for separating tiers of an application
  • Internet Gateway for public internet access
  • NAT Gateway for outbound internet access from private subnets
  • Security Groups and Network ACLs for traffic control
  • VPC Peering and Transit Gateway for connecting multiple VPCs
  • VPN and Direct Connect for hybrid cloud connectivity

Core Concepts

Subnets

A subnet is a range of IP addresses within a VPC. Public subnets have a route to an Internet Gateway and can communicate with the internet. Private subnets do not have direct internet access and are used for databases and application servers.

Route Tables

A route table contains rules (routes) that determine where network traffic is directed. Each subnet must be associated with a route table.

Internet Gateway (IGW)

An Internet Gateway allows resources in a public subnet to communicate with the internet. It is horizontally scaled and highly available.

NAT Gateway

A Network Address Translation (NAT) Gateway allows instances in a private subnet to initiate outbound connections to the internet without being directly accessible from the internet.

Security Groups

Security Groups act as virtual firewalls at the instance level. They are stateful — if inbound traffic is allowed, the response is automatically permitted.

Network Access Control Lists (NACLs)

NACLs are optional firewall rules at the subnet level. They are stateless — both inbound and outbound rules must explicitly allow traffic. Rules are evaluated in order by number.

Security Considerations

  • Place databases and application servers in private subnets
  • Use Security Groups for instance-level stateful filtering
  • Use NACLs for subnet-level stateless filtering as an extra layer
  • Use NAT Gateway for private subnet outbound access — never put databases in public subnets
  • Enable VPC Flow Logs to capture and monitor network traffic

Architecture Overview

VPC (10.0.0.0/16)
│
├── Public Subnet (10.0.1.0/24)
│       │
│       ├── Internet Gateway  ← internet traffic in/out
│       └── EC2 (web server)
│
└── Private Subnet (10.0.2.0/24)
        │
        ├── NAT Gateway       ← outbound internet only
        └── RDS (database)

Real-World Use Cases

  • Three-tier web application — web tier in public subnet, app and DB in private subnets
  • Hybrid cloud — connect on-premises data center to AWS via VPN or Direct Connect
  • Multi-account environments — use VPC Peering or Transit Gateway to connect VPCs
  • Compliance — isolate regulated workloads in dedicated VPCs

AWS Solutions Architect Exam Notes

  • A default VPC is created in every AWS Region automatically
  • Security Groups are stateful; NACLs are stateless
  • A subnet can only be associated with one route table at a time
  • Internet Gateway enables internet access for public subnets
  • NAT Gateway allows private instances to initiate outbound traffic — not inbound
  • VPC Peering is non-transitive — A↔B and B↔C does not mean A↔C
  • Each VPC has a maximum of 5 Elastic IPs per region by default

Key Takeaways

  • VPC provides an isolated network environment in AWS
  • Public subnets route to an Internet Gateway; private subnets use NAT Gateway for outbound
  • Security Groups (stateful, instance-level) and NACLs (stateless, subnet-level) layer security
  • Always put databases and sensitive workloads in private subnets
  • VPC Flow Logs help monitor and troubleshoot network traffic
